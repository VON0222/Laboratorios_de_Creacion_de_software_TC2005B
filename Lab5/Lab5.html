Preguntas:

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    2. Internet es la inmensa red de computadoras conectadas entre si alrededor de todo el mundo, mientras que la World Wide Web es la gran colección de paginas que se asienta sobre la red de computadoras.

  2. ¿Cuáles son las partes de un URL?

    3. Las partes de un URL son el protocolo, dominio, ruta y parametros.

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    4. GET: Solicitar datos de un recurso especifico donde no se modifican de ninguna manera. No cambia el estado del recurso.
    HEAD: Effecto similar a GET, pero lo que regresa HEAD no puede contener cuerpo. Sin embargo, lo que retorne si contendra los mismos encabezados que si se usara GET. Se puede utilizar para verificar si el recurso esta presente antes de realizar una salicitud GET.
    POST: Enviar datos a un servidor para crear un recurso.
    PUT: Actualizar un recurso existente en un servidor usando el contenido del cuerpo de la solicitud.
    PATCH: Aplicar modificaciones parciales a un recurso.
    DELETE: Borrar el recurso especificado.

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    5. El metodo POST para asi enviar información al servidor.

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    6. El metodo GET para obtener inforación de un recurso sin modificar.

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    7. Significa que el servidor HTTP nos devuelve un estatus ante una petición estandar correcta a la que puede responder sin problemas. No hay ningun error.

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    8. No, debido a que es un error originado de la petición, o en otras palabras un error por parte del cliente.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    9. Si, debido a que se trata de un error del servidor al tratar de cumplir con la solicitud, por lo que el desarrollador deber corregir el error del servidor.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    10. Significa que el uso de ese atributo ya no es recomendado, ya sea debido a que el estandar es actualizado o debido a que algunos tags o caracteristicas son agregadas, removidas o desaprovadas. Algunos ejemplos de elementos desaprobados en HTML5 son center, strike, tt, entre otros.

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    11. Las principales diferencias entre HTML4 y HTML5 son la compatibilidad con navegadores modernos, soporte multimedia, ya no se ajusta a SGML, mejor rendimiento y soporte movil, mejores controles de formularios y almacenamiento en la web.

  11. ¿Qué componentes de estructura y estilo tiene una tabla?

    12. Los componentes de estructura son celdas (td/th), filas (tr), grupos de filas (thead/tbody/foot), leyenda (caption) y referencia a las columnas (col y colspan).
    Los componentes de estilo son border de table (obsoleto), colspan y rowspan de td y th, span de col, span de colgroup.

  12. ¿Cuáles son los principales controles de una forma HTML5?

    13. Los principales controles de una forma en HTML5 son botones, checkboxes, radio buttons, menus, text input, file select, hidden controls y object controls.

  14. ¿Cuál es el ciclo de vida de los sistemas de información?

    15. Es el periodo de vida que tiene un sistema desde la concepción de la idea de crear u optimizar un sistema. Tiene 7 fases, planificación, análisis, diseño, desarrollo, integración y periodo de pruebas, implementación y mantenimiento.

  15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    16. Es el proceso de planificación, creación, pruebas y despliegue en un sistema de información. Tiene cuatro fases, planeación, diseño, desarrollo e implementación.
